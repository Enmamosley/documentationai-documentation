# Callouts

Usa Callouts para resaltar informacion critica sin romper el flujo de lectura.

## Tipos recomendados

<Callout kind="tip">
  Usa este tipo para buenas practicas operativas (por ejemplo, validaciones previas antes de enviar un permiso).
</Callout>

<Callout kind="info">
  Usa este tipo para contexto util: reglas del sistema, restricciones y comportamiento esperado.
</Callout>

<Callout kind="alert">
  Usa este tipo para bloqueos reales o acciones de alto impacto (cancelaciones, rechazos, inconsistencias de horario).
</Callout>

## Ejemplos del sistema

<Callout kind="info">
**Clasificacion manual por Atencion al Cliente**

Los permisos creados por Contacto requieren clasificacion de riesgo y validacion de actividad antes de aprobarse.
</Callout>

<Callout kind="tip">
**Minimo un tecnico**

Todo permiso debe incluir al menos un tecnico con nombre valido antes del envio.
</Callout>

<Callout kind="alert">
**Bloqueos de horario**

Si existe un bloqueo activo para el rango seleccionado, el permiso no puede enviarse hasta corregir fecha u hora.
</Callout>
