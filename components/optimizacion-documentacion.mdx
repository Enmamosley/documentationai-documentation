# Optimizacion de Documentacion

Esta guia define un patron simple para que todas las paginas del sistema se lean rapido, sean consistentes y reduzcan dudas operativas.

## Estructura recomendada por pagina

<Steps>
  <Step title="Abre con objetivo" icon="target">
    Inicia con 1-2 lineas que expliquen que resuelve la pagina y para que rol aplica.
  </Step>
  <Step title="Muestra el flujo" icon="list-ordered">
    Si hay secuencia, usa <code>{`<Steps>`}</code> para evitar bloques largos de texto.
  </Step>
  <Step title="Resalta riesgos" icon="alert-triangle">
    Usa <code>{`<Callout kind="alert">`}</code> solo para bloqueos reales o errores comunes.
  </Step>
  <Step title="Cierra con enlaces accionables" icon="arrow-right">
    Usa Cards para dirigir al siguiente paso del flujo.
  </Step>
</Steps>

## Componente correcto para cada caso

<Columns cols={3}>
  <Card title="Callout" icon="info">
    Para advertencias, notas clave y recomendaciones de seguridad.
  </Card>
  <Card title="Steps" icon="check-circle">
    Para procesos secuenciales (crear, revisar, aprobar, registrar).
  </Card>
  <Card title="Tabs" icon="layers">
    Para variantes por rol o escenario sin duplicar secciones.
  </Card>
</Columns>

## Plantilla base reutilizable

```mdx
# Titulo de la pagina

Resumen corto de objetivo y alcance.

<Callout kind="info">
Contexto importante que evita errores de interpretacion.
</Callout>

## Flujo

<Steps>
  <Step title="Paso 1" icon="play">Accion esperada.</Step>
  <Step title="Paso 2" icon="check">Validacion clave.</Step>
</Steps>

## Siguiente paso

<Columns cols={2}>
  <Card title="Pagina siguiente" href="/ruta/siguiente" icon="arrow-right">
    Continuar el flujo.
  </Card>
</Columns>
```

## Reglas de calidad

- Un objetivo por pagina.
- Maximo 1 alerta critica por seccion.
- Evitar parrafos largos; preferir listas o steps.
- Los enlaces deben apuntar a rutas internas existentes.
- Mantener terminologia estable: permiso, aprobacion, QR, dashboard, rol.

<Callout kind="tip">
Antes de publicar una pagina nueva, valida si sigue este patron. Si no, simplifica contenido antes de agregar mas texto.
</Callout>
